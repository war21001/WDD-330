Testing and Debugging

Errors, Exceptions, and Warnings

  • System Error - There's a problem with the system or external devices with which the program is interacting
  • Programmer Error - The program contains incorrect syntax or faulty logic; it could een be as simple as a type
    • UserError - The user entered data correctly, which the program is unable to handle
  • Exceptions - an error that produces a return value that can then be used by the program to deal wit the error
  • Stack Traces - an exception will produce. Sequence of functions/method calls that lead to the point where the error occurred.
  • Warnings - occurs if an error in the code that isn't enough to cause a program to crash

Importance of Testing and Debugging

  • write code to fail loudly

Strict Mode

  • produces more exceptions and warnings and prohibits the use of some deprecated features
  • help improve clarity and speed
  • 'use strict';
  • linting tools - JSLint, JSHint, Es lint
    • highlight sloppy programming practices
    • syntax errors
    • complain if certain style conventions are not followed
    • Style guide

Freature Detection

  • if statement to see if browser supports the method
  • modernizer
  • browswer sniffing

Debugging in the Browser

  • process of finding out where bugs occur in the code and then dealing with them
  • breakpoints
  • alert()
  • use the console - console.log, console.trace()
  • debugger keyword - make sure to take out

Error Objects

  • can be created by the host environment when an excetion occurs, or in the code with a constructor function -> const error = new Error();
  • EvalError, RangeError, Syntax Error, TypeError, URIError, InternalError
  • properties - name, message, error, stack
  • Throwing Exceptions
    • throw statement
    • throw an error object
    • catch block

Excetion Handling

  • when an error occurs, the program termeinates with an error message
  • try, catch, finally

Tests

  • can simply be a function that tests a piece of code runs as it should
  • Test-driven development (TDD) - process of writing tests before any actual code

WorkFlow

  1. write tests (that intially fail)
  2. write code to pass the tests
  3. refactor the code
  4. Test refactored code
  5. write more tests for new features
  • Testing frameworks - provide a structure to write meaningful test and then run them
  • Jest
