Validating Forms

Client-Side Form Validation

  • Before submitting data to the server, it is important to make sure all required forms inputs are filled out, in the correct format
  • initial check, good user experience - noticable delay if it gets to the server a then rejected
  • Whatever you do -DO MOBILE FIRST - start small and scale up
  • not exhaustive security measure.

What is form Validation?

  • required fields
  • specific format is required
  • data entered must be in a specific format or specific way
  • browser/web server will check to see the datat is in the correct format and within the constraints set by the application

Why Validate?

  • get right data, in the right format
  • protect user's data
  • protect ourselves
  • never trust data from a user - a malicious user can still alter the network request

Types of form Validation - Client-Side

  • Built in form validation - use HTML5 built form validation features
  • JavaScript - validation is coded using JavaScript. Completely customizable but must create it or use a library

Built in form Validation

  • use validation attributes on form elements
  • required
  • minlength and maxlength
  • min and max --numbers
  • type
  • pattern -- Regular Expressions
  • CSS-psuedo class :valid or :invalid
  • Regular Expression- pattern that can be used to match character combinations in text strings

Validating Forms using JavaScript

  • Constraint Validation API - set of methods and properties available
  • HTMLButtonElement, HTMLFieldSetElement, HTMLInputElement, HTMLOutputElement, HTMLSelectElement, HTMLTextAreaElement
  • Properties -- ValidationMessage, validility, willValidate
  • Methods -- checkValidity(), reportValidility(), preventValidility(), setCustomValidility()

Questions to consider when not using a built-in API

  • What kind of validation should I perform?
  • What should I do if the from doesn't validate?
  • How can I help the user to correct invalid data?
  • Provide as much helpful information as possible to guide the user

Using Fetch

FetchAPI provides a JavaScript interface for accessing and maniputlating parts of the HTTP pipeline, sucha as requests and responses

  • global fetch() method -- previously XMLHTTPRequest
  • example:
    fetch('http://example.com/movies.json')
                .then(response => response.json())
                .then(data => console.log(data));
  • fetch a JSON file across the network - takes one parameter, the path to the resource - returns a promise that resolves with a Response Object
  • The Reponse Object, does not directly contain the actual JSON response body but instead a representation of the entire HTTP response.
  • to extract the JSON Body Content from the Response Object using json() method that returns a second promise that resolves with the result of parsing the reponse body text as JSON

Supplying Request options

  • Optional second parameter -- init
  • credentials: 'include'
